# exam-note 스킬

합격률 최우선 강사 관점으로, 기출 JSON 데이터를 분석해 시험에 나오는 것만 추린 핵심 요약 노트를 생성한다.

---

## ⛔ 절대 규칙 (위반 시 작업 즉시 중단)

| 번호 | 규칙 |
|------|------|
| 1 | `src/data/exam/` 하위 JSON 파일은 **절대 수정하지 않는다.** `Read` 툴로만 읽는다. `Edit`·`Write` 툴 사용 금지. |
| 2 | 오직 `.note/` 경로의 파일만 생성·수정할 수 있다. |
| 3 | 기출 원문에 없는 개념·선지·학자명을 창작하지 않는다. 노트 내용은 100% 실제 기출 데이터 기반이어야 한다. |
| 4 | 연도별 기출 파일, 원본 데이터 복사본 등 중복 산출물을 생성하지 않는다. |

---

## 입력 파악

사용자 요청에서 다음을 확인한다:

- **과목 코드**: SSO, SET, EPH, SPS, KHS (없으면 질문)
- **연도 범위**: 기본값 = 현재 기준 최근 7개년
- **데이터 경로**: 기본값 = `src/data/exam/`
- **출력 경로**: 기본값 = `.note/`
- **출력 파일명**: `.note/{과목코드}_핵심요약노트.md` (기존 파일 존재 시 버전 관리 규칙 적용)

JSON 파일 이름 규칙: `{SUBJECT}_{YEAR}_sports_instructor_exam.json`

---

## 실행 순서

### 1단계: 데이터 수집

각 연도 JSON을 `Read` 툴로 로드한다. 파일이 없거나 파싱 오류가 발생하면 사용자에게 알리고 해당 파일만 건너뛴다.

각 문제 구조:
```
id, question, view, options[], answer, explanation
```
- `answer`: `number` (단일) 또는 `number[]` (복수정답)

### 1.5단계: view 전처리

`view`를 무시하면 토픽 분류 정확도와 함정 패턴 품질이 크게 떨어진다.

| view.type | 처리 방법 |
|-----------|-----------|
| `NONE` | question + options + explanation만 사용 |
| `PASSAGE` | `view.passage[]{text, underline}`를 question 앞에 붙여 분석. 지문이 토픽 분류의 1순위 근거 |
| `ITEMIZED` | `view.items[]{label, content[]}`를 함께 읽어 정답 매칭. 오답 조합 전체를 오답 데이터로 수집 |
| `BLANK` | 빈칸 위치와 정답 선지를 연결해 "빈칸 = 핵심 암기 개념"으로 처리 |
| `VISUAL` | `view.media.alt`와 explanation 위주로 분석. 이미지 자체는 분석 불가 |
| `COMPOSITE` | `view.passage` + `view.items` 모두 수집 |
| `TABLE` | `view.table.headers`와 `view.table.rows`의 셀 내용을 행·열 순서로 읽어 텍스트화. `colSpan`/`rowSpan`이 있으면 병합 구조를 고려해 의미 단위로 해석 |

### 2단계: 문제 분류 — 반복출제 vs 변별력

모든 문제를 두 그룹으로 나눈다.

**반복출제 문제** (핵심 노트 재료)
- 3개년 이상 동일하거나 유사한 개념·키워드로 출제된 문제
- 정답 선지가 안정적으로 수렴하는 문제 (매년 같은 이론·학자·개념이 정답)

**변별력 문제** (고급 노트 재료)
- 1~2개년만 출제된 개념
- 세부 수치·예외 케이스·비교 조합을 묻는 고난도 문제
- ITEMIZED / BLANK 유형 중 선지 조합이 교묘한 문제
- 정답률이 낮을 것으로 추정되는 문제 (오답 선지가 매우 유사한 경우)

### 2.5단계: explanation 활용

`explanation` 필드가 존재하는 문제는 아래 기준으로 노트 내용에 반영한다.

| explanation 필드 | 노트 반영 방법 |
|-----------------|--------------|
| `correct` | 해당 토픽의 개념 설명 근거로 우선 사용. 표 셀이나 서술에 직접 인용 가능 |
| `distractors[].term` + `.reason` | 반복출제 문제라면 ⚠️ 주의 박스에, 변별력 문제라면 섹션 3 ❌ 함정에 활용. 문제 분류(2단계)가 기준 |
| `summary` | 비교형·개념-설명형 표의 정리 행 또는 토픽 마무리 서술에 반영 |

`explanation`이 없는 문제는 question + view + options만으로 개념을 추출한다. 내용을 창작하지 않는다.

### 3단계: 챕터 추론 및 사용자 확인 (KHS 제외)

**KHS 과목**은 이 단계를 건너뛴다. KHS는 시대 단위로 처리한다 (6단계 참조).

**그 외 과목**:

#### 3-1. 챕터 귀납 추론

모든 과목에 동일한 귀납 방식을 적용한다. 각 문제의 `question`, `view`, `options`, `explanation.summary`를 종합해 챕터를 배정한다.

- 개론 성격의 챕터 (스포츠사회학의 정의, 연구영역, 발전 요인 등 기초 개념)는 **Chapter 1**로 고정
- 나머지 챕터는 아래 3-2 기준으로 번호를 부여한다
- 챕터명은 교재 단원명에 준하는 정형화된 이름으로 짓는다 (예: "스포츠와 정치", "스포츠 사회화")
- 토픽명도 동일하게 문제에서 귀납 추론한다

#### 3-2. 챕터 순서 결정 (중앙값 방식)

1. 각 챕터에 속한 문제들의 **문제 번호를 7개년 전체에서 수집**한다
2. 해당 번호들의 **중앙값을 평균** 낸다
3. 중앙값 평균 오름차순으로 챕터 번호를 부여한다 (Chapter 1 제외)

예시:
```
스포츠 사회화 → 7개년 문제 번호 [4,4,3,5,4,3,4] → 중앙값 평균 ≈ 3.9 → Chapter 2
스포츠와 정치 → 7개년 문제 번호 [6,12,7,6,8,7,6] → 중앙값 평균 ≈ 7.1 → Chapter 3
스포츠와 계층 → 7개년 문제 번호 [11,20,13,11,12,11,14] → 중앙값 평균 ≈ 13.1 → Chapter 4
```

#### 3-3. 사용자 확인

노트 생성 전 반드시 아래 형식으로 챕터 목록을 제시하고 **사용자 승인을 받는다**. 승인 전에는 노트를 생성하지 않는다.

```
아래 챕터 구성으로 노트를 생성하겠습니다. 확인 후 진행 여부를 알려주세요.

Chapter 1. [챕터명] — 토픽: [토픽1], [토픽2], ...
Chapter 2. [챕터명] — 토픽: [토픽1], [토픽2], ...
...
```

사용자가 챕터명·토픽명·순서를 수정하면 반영 후 노트 생성을 진행한다.

### 4단계: 문제 분류 — 반복출제 vs 변별력 (챕터·토픽 단위)

2단계에서 분류한 결과를 챕터·토픽 단위로 재정리한다.

### 5단계: 노트 생성

#### 파일 버전 관리

노트 생성 전 `.note/` 경로에 동일 과목 노트가 존재하는지 확인한다.

**기존 파일 없음**: `.note/{과목코드}_핵심요약노트.md` 로 생성

**기존 파일 있음**: 버전 넘버링 방식으로 처리
1. 기존 파일의 현재 버전 번호를 파악한다
   - `{과목코드}_핵심요약노트.md` → 버전 미부여 상태 = v1으로 간주
   - `{과목코드}_핵심요약노트_v2.md` → 현재 최신 버전 = v2
2. 기존 최신 파일을 `_v{N}`으로 rename한다 (`Bash` 툴 사용)
   - 예: `SSO_핵심요약노트.md` → `SSO_핵심요약노트_v1.md`
   - 예: `SSO_핵심요약노트_v2.md` → `SSO_핵심요약노트_v2.md` (이미 버전 있으면 그대로)
3. 새 파일을 `_v{N+1}`으로 생성한다
   - 예: `SSO_핵심요약노트_v2.md`

버전 파악 예시:
```
기존: SSO_핵심요약노트.md → rename → SSO_핵심요약노트_v1.md, 신규: SSO_핵심요약노트_v2.md
기존: SSO_핵심요약노트_v2.md → 그대로 유지, 신규: SSO_핵심요약노트_v3.md
```

### 6단계: KHS 처리 (KHS 전용)

KHS는 챕터 대신 **시대 단위**로 구성한다. 시대 오름차순 정렬:
고대 → 삼국·통일신라 → 고려 → 조선 → 개화기·근대 → 일제강점기 → 광복 이후·현대

---

## 출력 구조

```
# {과목명} 핵심 요약 노트
### 스포츠지도사 2급 | {연도범위} 기출 분석 기반

> 강사 한마디: ...
```

`강사 한마디`는 해당 과목의 출제 특성과 수험 전략을 한 문장으로 압축한다.
예시: `"이 과목은 학자명과 이론 이름을 세트로 암기하는 게 핵심입니다. 개념보다 이름이 먼저 떠올라야 합니다."`

---

### 섹션 1: 빈출 토픽 현황

챕터 헤더 아래 토픽 행이 들어가는 단일 테이블 구조. 챕터 순서는 섹션 2와 동일.

빈도는 🔥 마커로 표시한다:
- 🔥🔥🔥 = 5개년 이상 출제
- 🔥🔥 = 3~4개년 출제
- 🔥 = 1~2개년 출제 (변별력 문제 위주)

| 챕터 | 토픽 | 빈도 | 반복출제 | 변별력 | 출제 연도 |
|------|------|------|----------|--------|-----------|
| **Chapter 1. [챕터명]** | | | | | |
| | [토픽명] | 🔥🔥🔥 | 3 | 1 | 2019, 2020, ... |
| | [토픽명] | 🔥🔥 | 2 | - | 2021, 2022 |
| **Chapter 2. [챕터명]** | | | | | |
| | [토픽명] | 🔥🔥 | 2 | 1 | 2019, 2021 |

> `반복출제` / `변별력` 컬럼에는 해당 그룹의 **문제 수**를 숫자로 기입한다. 없으면 `-`.

---

### 섹션 2: 핵심 노트 (반복출제 문제 기반)

**반복출제 문제만** 사용. 챕터 → 토픽 계층 구조로 정렬.

챕터 헤더:
```
# Chapter [number]. [챕터명]
```

토픽 헤더 (반복출제 = 3개년 이상이므로 🔥🔥 이상만 가능):
```
## [토픽명] · 🔥🔥🔥   ← 5개년 이상
## [토픽명] · 🔥🔥     ← 3~4개년
```

챕터당 토픽이 1개뿐이어도 챕터 → 토픽 계층 구조를 그대로 유지한다.

**KHS**: 시대별 사건·제도·인물을 시간 흐름에 따라 서술한다. 표보다 서술+불릿이 적합한 경우가 많다.

**그 외 과목**: 아래 표 유형 카탈로그에서 토픽 성격에 맞는 유형을 선택한다. 하나의 토픽 안에서 여러 유형을 조합해도 된다.

---

#### 표 유형 카탈로그

**[학자-이론형]** — 특정 학자의 이론·모형이 반복 출제되는 토픽
| 학자명(영문) | 이론·모형명 | 핵심 키워드 |
|-------------|------------|------------|

**[개념-설명-예시형]** — 개념 정의와 적용 사례를 함께 묻는 토픽
| 개념 | 설명 | 기출 예시 |
|------|------|----------|

**[비교형]** — 두 개념·이론·제도를 구분하는 것이 핵심인 토픽
| 구분 | A | B |
|------|---|---|

**[단계-순서형]** — 단계·절차·발달 순서를 묻는 토픽
| 단계 | 명칭 | 내용 |
|------|------|------|

**[목록형]** — 표로 만들기 어색하고 단일 사실 나열이 더 명확한 경우
- 불릿으로 나열

---

이론가 이름은 항상 한글(영문) 병기: `코클리(Coakley)`
같은 이론가가 여러 토픽에 등장하면 `→ [토픽명] 참조` 로 상호 참조한다.
실제 선지 원문을 직접 인용해 실전감을 유지한다.

각 토픽 끝:
```
> ⚠️ **주의** {반복출제 문제에서 나온 함정만. 자주 반복되는 오답 패턴 1~2개}
```

> **⚠️ 작성 기준**: 반복출제 문제(섹션 2 재료)에서 나온 함정만 여기에 쓴다. 변별력 문제에서 나온 함정은 ⚠️에 쓰지 않고 섹션 3에만 쓴다.

---

### 섹션 3: 고급 노트 (변별력 문제)

**변별력 문제만** 사용. 챕터 → 토픽 계층 구조로 정렬. **챕터 순서는 섹션 2와 동일.**

챕터 헤더:
```
# Chapter [number]. [챕터명]
```

각 항목은 `### {개념명}` 헤더로 시작한다. 개념 설명은 섹션 2와 동일한 **표 유형 카탈로그**를 적용한다. 변별력 문제의 특성상 **비교형**과 **단계-순서형**이 자주 적합하다.

챕터당 토픽이 1개뿐이어도 챕터 → 토픽 계층 구조를 그대로 유지한다.

설명 후 반드시 함정 항목을 추가한다:
```
❌ {실제 오답 선지 원문} → {왜 틀렸는지 한 줄}
```

`explanation.distractors`가 있으면 `term`과 `reason`을 함정 항목에 직접 활용한다.

> **❌ 작성 기준**: 변별력 문제(섹션 3 재료)에서 나온 함정만 여기에 쓴다. 반복출제 문제의 함정은 섹션 2 ⚠️에만 쓴다.

---

## 품질 기준

- 이론가 이름: 항상 한글(영문) 병기
- 함정 선지: 실제 오답 선지 원문 기반 (창작 금지)
- 노트 생성 후 자가 점검:
  - [ ] JSON 파일을 수정하지 않았는가?
  - [ ] 창작 개념이 없는가? (100% 기출 기반)
