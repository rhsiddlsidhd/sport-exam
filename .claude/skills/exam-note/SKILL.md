# exam-note 스킬

합격률 최우선 강사 관점으로, 기출 JSON 데이터를 분석해 시험에 나오는 것만 추린 핵심 요약 노트를 생성한다.

## 입력 파악

사용자 요청에서 다음을 확인한다:

- **과목 코드**: SSO, SET, EPH, SPS, KHS 등 (없으면 질문)
- **연도 범위**: 기본값 = 현재 기준 최근 7개년
- **데이터 경로**: 기본값 = `src/data/exam/`
- **출력 경로**: 기본값 = `.note/`

JSON 파일 이름 규칙: `{SUBJECT}_{YEAR}_sports_instructor_exam.json`

## 실행 순서

### 1단계: 데이터 수집 및 검증

각 연도 JSON을 로드한다. 파일이 손상된 경우(JSON 파싱 오류) 즉시 수정 후 계속한다.
각 문제 구조: `id`, `question`, `view`, `options[]`, `answer`, `explanation`
- `answer`는 `number` (단일정답) 또는 `number[]` (복수정답) 두 가지 형태
- `view`는 타입에 따라 사용하는 필드가 다름 (아래 참조)

### 1.5단계: view 데이터 전처리

`view.type`별로 분석에 사용할 텍스트와 정답 데이터를 다르게 처리한다.
전체 문제의 약 60%가 view를 가지며, 이를 무시하면 토픽 분류 정확도와 함정 패턴 품질이 크게 떨어진다.

| view.type | 사용 필드 | 처리 방법 |
|-----------|-----------|-----------|
| `NONE` | — | question + options + explanation만 사용 |
| `PASSAGE` | `view.passage: string[]` | 지문 배열을 question 앞에 붙여 분석 텍스트로 사용. 지문이 토픽 분류의 1순위 근거 |
| `ITEMIZED` | `view.items: {label, content}[]` | label(㉠㉡㉢)과 content를 함께 읽어야 정답 매칭 의미 파악 가능. 정답 선지("㉠-혁신주의/㉡-의례주의")와 items를 대응시켜 개념 추출 |
| `BLANK` | `view.passage: string[]` | 빈칸(괄호) 위치와 정답 선지를 연결해 "빈칸에 들어갈 개념"을 핵심 암기 포인트로 처리 |
| `VISUAL` | `view.media: {type, url, alt}` | 이미지/차트 기반 문제. alt 텍스트와 explanation으로 개념 추출. 이미지 자체는 분석 불가이므로 explanation 비중을 높임 |
| `COMPOSITE` | `view.passage` + `view.items` 혼합 | 두 필드 모두 수집해 분석 텍스트에 포함 |

PASSAGE/ITEMIZED/BLANK/VISUAL 문제는 "지문 기반 유형"으로 태깅해두면
섹션 4(최근 출제 트렌드)에서 유형 변화 분석에 활용할 수 있다.

### 2단계: 토픽 분류 및 빈도 분석

Python 스크립트를 활용해 전처리된 전체 텍스트(question + view 각 필드 + options + explanation)를 기반으로 토픽을 분류한다.
토픽 키워드는 과목 특성에 맞게 설정하되, 다음 원칙을 따른다:

- 8~12개 토픽으로 분류 (너무 세분화하면 패턴이 안 보임)
- 출제 빈도 + 출제 연도를 함께 집계
- 오답 선지 수집 시 `answer`가 `number[]`인 복수정답 문제는 정답 아닌 모든 option을 오답으로 처리
- ITEMIZED 유형은 오답 매칭 조합 전체를 오답 데이터로 수집 → 헷갈리는 조합 패턴 추출

### 3단계: 노트 생성

`.note/{SUBJECT}_핵심요약노트.md` 파일을 생성한다.

**절대 생성하지 않는 파일**: 연도별 기출 파일, 원본 데이터 복사본 등 중복 산출물

## 출력 구조 (이 순서와 형식을 유지)

```
# {과목명} 핵심 요약 노트
### {자격증명} | {연도범위} 기출 분석 기반

> 강사 한마디: ...
```

### 섹션 1: 빈출 토픽 순위표
출제 횟수 내림차순 표. 컬럼: 순위 / 토픽 / 출제 횟수 / 출제 연도

### 섹션 2: 토픽별 핵심 개념 (출제 빈도 상위 순으로)
각 토픽마다:
- 개념 설명은 표 형식 우선 (이론가 이름 + 개념 반드시 세트)
- 실제 선지 원문을 직접 인용해 실전감 유지
- 교과서 설명 금지 — 시험에 나온 내용만

### 섹션 3: 함정 선지 총정리
오답 선지 중 반복 등장하거나 헷갈리기 쉬운 것을 추출.
형식: `❌ 틀린 선지` → 올바른 내용 한 줄 설명

### 섹션 4: 최근 3년 신출제 경향
연도별로 새롭게 등장했거나 심화된 유형 분석.
마지막에 트렌드 요약 한 줄.

### 섹션 5: 압축 암기 공식
코드블록 안에 핵심만 짧게. 예:
```
이론명 = 요소1·요소2·요소3
학자명 단계 = A→B→C→D
```

## 품질 기준

- 이론가 이름은 항상 한글(영문) 병기: `코클리(Coakley)`
- 같은 이론가가 여러 섹션에 등장할 때 설명을 반복하지 않고 상호 참조
- 함정 선지는 실제 오답 선지 원문 기반 (지어내지 말 것)
- 암기 공식은 5개 이상 10개 이하로 압축
