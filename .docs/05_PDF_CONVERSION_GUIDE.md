# 04. PDF Conversion Guide

## 개요

PDF 시험지를 `ExamQuestion` JSON 구조로 변환할 때, AI가 구조 판단에 어려움을 겪는 케이스를 사례별로 기록한다.
각 사례는 **잘못된 변환 → 올바른 변환** 형태로 제시하며, 판단 근거를 함께 명시한다.

---

## 보기(view) 구조 판단

### CASE 01 — 대화문은 `list`로 변환한다

**상황**: 보기가 화자(A, B, C 등) 간의 대화 형식으로 구성된 경우

**❌ 잘못된 변환** — `text`에 평문으로 나열

```json
"view": {
  "text": [
    "A: 오늘 테니스 경기 봤어? 서브권이나 코트 선택은 어떻게 정해?",
    "B: 응, 보통 동전 던지기나 라켓 돌리기로 정하더라.",
    "C: 그래서 테니스는 운도 실력이라고 하잖아.",
    "A: 그러면 시합을 시작할 때 항상 저런 방식으로 정하는 거야?",
    "C: 동전 던지기로 정하는 경우가 많아"
  ]
}
```

**✅ 올바른 변환** — `list`에 label/content 구조로 분리

```json
"view": {
  "list": [
    { "label": "A", "content": ["오늘 테니스 경기 봤어? 서브권이나 코트 선택은 어떻게 정해?"] },
    { "label": "B", "content": ["응, 보통 동전 던지기나 라켓 돌리기로 정하더라."] },
    { "label": "C", "content": ["그래서 테니스는 운도 실력이라고 하잖아."] },
    { "label": "A", "content": ["그러면 시합을 시작할 때 항상 저런 방식으로 정하는 거야?"] },
    { "label": "C", "content": ["동전 던지기로 정하는 경우가 많아"] }
  ]
}
```

**판단 근거**:
- 화자가 구분되는 대화문은 `label`(화자)과 `content`(발화)의 구조가 명확히 존재한다.
- `text`는 순수 지문(단일 화자 없는 서술)에만 사용한다.
- 한 화자가 여러 번 등장해도 각 발화를 별도 `list` 항목으로 분리한다.

**출처**: `SET_2025` Q7

---

### CASE 02 — 빈칸 기호는 반드시 괄호로 감싸 `blanks`에 넣는다

**상황**: 보기 지문에 `㉠`, `㉡` 같은 빈칸 기호가 괄호 없이 텍스트 중간에 삽입된 경우

**❌ 잘못된 변환** — `text`에 괄호 없이 그대로

```json
"view": {
  "text": [
    "배구 경기의 주심인 ㉠ A 심판은 네트 위 높이 약 3.43m에서 경기를 주관하고,",
    "부심인 ㉡ B 심판은 주심 맞은편에서 네트 폴 옆에 위치하여 경기를 지원한다."
  ]
}
```

**✅ 올바른 변환** — `blanks`에 괄호 `(㉠)` 형태로

```json
"view": {
  "blanks": [
    "배구 경기의 주심인 (㉠) A 심판은 네트 위 높이 약 3.43m에서 경기를 주관하고,",
    "부심인 (㉡) B 심판은 주심 맞은편에서 네트 폴 옆에 위치하여 경기를 지원한다."
  ]
}
```

**판단 근거**:
- `blanks` 키는 빈칸 채우기 형식의 지문에 사용한다.
- UI 렌더러(`renderTextWithHighlight`)는 `( ㉠ )` 또는 `(㉠)` 패턴을 감지해 강조 처리한다. 괄호가 없으면 강조가 적용되지 않는다.
- PDF에서 괄호가 생략된 채 인쇄된 경우에도 변환 시 `(㉠)` 형태로 보정한다.
- `text`는 빈칸 기호가 전혀 없는 순수 서술 지문에만 사용한다.

**출처**: `SET_2025` Q13

## view 키 선택 기준 요약

| 보기 유형 | 사용할 키 | 조건 |
|-----------|-----------|------|
| 순수 서술 지문 | `text` | 화자 구분 없음, 빈칸 기호 없음 |
| 빈칸 채우기 지문 | `blanks` | `(㉠)`, `( ㉡ )` 등 패턴 포함 |
| 밑줄 지문 | `text` + `underlines` | 밑줄 표시할 줄의 인덱스를 `underlines`에 명시 |
| 대화문 (화자 구분) | `list` | A/B/C 등 화자 기반 |
| 기호 항목 나열 | `list` | ㄱ, ㄴ, ㄷ 또는 ㉠, ㉡ 등 기호 기반 항목 |
| 이미지·도표 | `media` | 이미지 파일 필요 |
| 표 형식 | `table` | 행·열 구조 |

> `text`와 `blanks`는 같은 `view` 안에 공존할 수 있다.
> 예: 서술 지문 + 빈칸 지문이 함께 있는 경우 `text`와 `blanks` 모두 작성.

---

### `text` — 순수 서술 지문

```json
"view": {
  "text": [
    "운동훈련에 의한 효과는 운동량이 일상생활 수준보다 높을 때 일어난다.",
    "운동량은 운동의 빈도, 강도 또는 지속시간을 증가시킴으로써 늘릴 수 있다."
  ]
}
```

---

### `blanks` — 빈칸 채우기 지문

빈칸 기호는 반드시 `(㉠)` 형태로 괄호를 포함해야 한다.

```json
"view": {
  "blanks": [
    "자율신경계는 '흥분성'의 ( ㉠ )과 '억제성'의 ( ㉡ )으로 구분된다."
  ]
}
```

---

### `text` + `underlines` — 밑줄 지문

`underlines`는 `text` 배열의 인덱스(0-based)를 지정한다.

```json
"view": {
  "text": [
    "A 선수가 패배해야 할 상황이었지만 심판은 감점을 선언하지 않았다.",
    "결국 A 선수는 승리했지만, 부끄러운 마음에 조용히 집으로 돌아갔다."
  ],
  "underlines": [1]
}
```

---

### `list` — 대화문 (화자 구분)

화자(A, B, C 등)가 구분되는 대화. 한 화자가 여러 번 등장해도 각 발화를 별도 항목으로 분리한다.

```json
"view": {
  "list": [
    { "label": "A", "content": ["오늘 테니스 경기 봤어? 서브권은 어떻게 정해?"] },
    { "label": "B", "content": ["보통 동전 던지기로 정하더라."] },
    { "label": "A", "content": ["그러면 항상 저런 방식으로 정하는 거야?"] },
    { "label": "B", "content": ["동전 던지기로 정하는 경우가 많아."] }
  ]
}
```

---

### `list` — 기호 항목 나열

ㄱ/ㄴ/ㄷ, ㉠/㉡/㉢ 등 기호로 구분된 항목 나열.

```json
"view": {
  "list": [
    { "label": "ㄱ", "content": ["세 참가자의 분당환기량은 동일하다."] },
    { "label": "ㄴ", "content": ["다영의 폐포 환기량은 분당 6L/min이다."] },
    { "label": "ㄷ", "content": ["지현의 해부학적 사강은 150mL이다."] }
  ]
}
```

---

### `media` — 이미지·도표

> **⚠️ 클라이언트(작업자) 안내**: `media` 타입 문제는 이미지 파일을 별도로 준비해야 합니다.
> JSON 변환만으로는 완료되지 않으며, 이미지 추출 → 파일 배치 작업이 추가로 필요합니다.

**처리 방식 — 2단계 분리**

| 단계 | 작업 | 담당 |
|------|------|------|
| 1단계 | JSON 변환 시 URL을 네이밍 규칙대로 미리 채워둠 | AI 변환 |
| 2단계 | PDF에서 이미지를 추출해 `public/exam/`에 파일 배치 | 작업자 직접 |

**1단계에서 URL을 미리 확정해두는 이유**:
- 이미지가 아직 없어도 JSON 구조는 완성된 상태로 유지된다.
- 나중에 이미지 파일만 올바른 경로에 배치하면 별도 수정 없이 연결된다.
- `/exam/` 경로 목록과 `public/exam/` 실제 파일을 비교하면 누락된 이미지를 추적할 수 있다.
- 이미지 로드 실패 시 `alt` 텍스트로 fallback되므로 앱이 깨지지 않는다.

**네이밍 규칙**: `{SUBJECT}-{YEAR}-{QuestionNumber}.webp`

```json
"view": {
  "media": {
    "type": "IMAGE",
    "url": "/exam/EPH-2024-17.webp",
    "alt": "산소-헤모글로빈 해리 곡선의 운동 시 변화"
  }
}
```

---

### `table` — 표 형식

2단 헤더의 경우 `rowSpan: 2`인 셀이 있으면 다음 헤더 행의 `cells`에서 해당 열을 생략한다.

```json
"view": {
  "table": {
    "caption": "기관별 기능 여부",
    "headers": [
      {
        "cells": [
          { "content": "기능", "rowSpan": 2, "isHeader": true },
          { "content": "기관", "colSpan": 3, "isHeader": true }
        ]
      },
      {
        "cells": [
          { "content": "뇌하수체", "isHeader": true },
          { "content": "부신", "isHeader": true },
          { "content": "㉠", "isHeader": true }
        ]
      }
    ],
    "rows": [
      {
        "cells": [
          { "content": "체액량 조절 호르몬 분비" },
          { "content": "㉡" },
          { "content": "○" },
          { "content": "×" }
        ]
      }
    ]
  }
}
```
